# 🧠 personal-context-repo

English version → [README.md](README.md)

一个自初始化的个人上下文系统，为 AI 协作而设计。

## 这是什么

每次和 AI 对话都要重新解释自己是谁、在做什么、上次聊到哪了。`personal-context-repo` 解决这个问题。你把一个文件（`AGENTS.md`）放进任意目录，开始和 AI 助手对话，它会根据你的真实情况、用你的语言，自动构建一个结构化的上下文仓库。

初始化完成后，同一个文件继续管理日常的内容路由、知识溯源、落盘规则。它不只是给 AI 加记忆——它记录你的项目、决策、悬而未决的问题、不断演化的想法，以及这些想法是怎么一步步变成现在这样的。

## 🚀 怎么用

1. 把 `AGENTS.md` 放进一个空目录，或者你已有的项目目录
2. 用 AI 助手打开这个目录开始对话——它会检测到仓库未初始化，问你几个问题
3. AI 自动生成完整结构，用你的语言，填入你的真实上下文

## 📁 生成的结构

```text
your-repo/
├── AGENTS.md           # AI 日常工作规则
├── INIT.md             # 一次性初始化指南（读一次后不再加载）
├── CONVENTIONS.md      # 维护规范（自动生成）
├── current.md          # 当前状态快照
├── me/
│   └── profile.md      # 个人档案
├── journal/            # 每日想法与笔记
├── projects/           # 项目与想法
├── references/         # 外部资料
└── skills/             # 可选的 AI 工作流
```

## 💡 实际效果

### 知识溯源标记 — 分清谁想的

每条写入仓库的洞察都会标记来源：

```markdown
真正的瓶颈不是 AI 的能力，而是人的注意力。
当你的提问很模糊时，AI 会用大量新概念来补偿——
结果你看完之后比问之前更困惑。
<!-- origin: co-created | 从"为什么 AI 没有让学习变快"出发，
     发现问题是注意力过载，不是信息不够 -->

这意味着产品应该在 AI 回答之前先帮用户把问题想清楚，
而不是在回答之后加检查点。
<!-- origin: self -->

可以考虑脚手架模式：不阻止用户直接问 AI，
而是设计一个自然引导用户先思考的工作流。
<!-- origin: ai-generated -->
```

三个月后回看，你仍然能分清哪些判断是自己想通的，哪些是 AI 建议的，哪些是讨论中碰撞出来的。

### 跨会话收束 — 想法越聊越清晰

同一个主题在多次对话中逐步演化，每一步都有记录：

```text
3月15日 — "也许产品应该在 AI 完成任务后加一个认知检查点"

3月22日 — "事后检查点不管用——用户已经进入被动接收状态了。
           应该在源头介入：先帮用户把问题想清楚，再让 AI 回答。"

4月03日 — "这不只是关于提问。而是帮人在 AI 能生成一切的时代，
           维护属于自己的坐标系——品味、判断力、意愿。"

4月17日 — "不要急着定义产品。先定义一种世界观：
           当 AI 极其强大时，人和 AI 的关系应该是什么样的？"
```

每次新对话从上次的进度继续推进，而不是从零开始。

## ⚔️ 为什么不用 Mem0、Zep 这些 Agent Memory

Mem0、Zep、Graphiti、Letta、Supermemory——都承诺自动跨会话记忆。实际上，自动提取产出的是低质量、不透明、经常出错的上下文。一个被广泛引用的 Mem0 生产环境审计显示，运行 32 天后，**97% 的自动提取记忆是垃圾**——重复条目、提示词碎片、幻觉放大的偏好、各种噪音。

更深的问题是失控。你看不到系统记住了什么，没法方便地修正，没法按自己的逻辑组织。所谓的"自动化"反而成了最难用的部分。

`personal-context-repo` 走相反的路：没有自动提取，你自己写下重要的东西，存在你能直接打开和编辑的纯文本文件里。代价是花一点时间维护——但远低于反复纠正错误的自动记忆，或者更糟：你根本没发现它记错了。

## ✨ 核心特性

- 🔄 **自初始化** — AI 帮你搭建，无需手动配置
- 🌍 **语言自适应** — 支持任何语言
- 👤 **人工策展** — 你控制 AI 知道什么
- 🏷️ **知识溯源标记** — 追踪想法来自你、AI、还是共同讨论
- 📝 **即时落盘** — 重要洞察在对话中保存，而非对话后
- 📂 **纯 Markdown + Git** — 透明、可移植、有版本历史
